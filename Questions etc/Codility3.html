Task description

The objective of this task is to finish the implementation of UsersListViewModel by providing code for the fetchUsers() method. UsersListViewModel will be responsible for fetching a list of users and returning them in the correct order.

Requirements

In order to implement the fetchUsers() method, the following requirements must be fulfilled:

  1. To fetch the users, use usersService by calling the fetchUsers method. - If the fetch fails, display an alert message using alertPresenter passed in the UsersListViewModel initializer. As you might have noticed, the alert presenter function

     `presentDecisionAlert(title: String, message: String, onYes: @escaping () -> (), onNo: @escaping () -> ())`

 can handle two paths, `onYes` and `onNo`:

     - For the `onYes` path, please retry the fetch request.
     - For the `onNo` path, please close the users list using one of the `delegate` methods.

 - For the `title` and `message`, use `errorTitle` and `errorMessage` (available in `UsersListViewModel`).
 Please notice that `errorMessage` should be created with the `error code` that can be retrieved from `error` in the `.failure(let error)` case.

  2. If the result of fetchUsers is retrieved successfully, then please return an alphabetically sorted list of users by calling one of the delegate methods.

  3. Please remember about the completion of updateLoadingState; use it to inform the user about fetch progress. Use .loading during the fetch request and .normal once the request is complete.

Please be careful not to cause memory leaks.

Assumptions

  • Use Swift 5.3.
  • Use the Foundation framework; documentation can be found here.

Hints

  • take special care to check Protocols below. Especially AlertPresenter has some limitations.

Protocols and structures

Below, you can find the protocols and structures that are used in UsersListViewModel. They will be useful if you want to work in a separate Xcode project or playground in order to provide a solution.

Please do not change the following structures and protocols. Provide your implementation only within the fetchUsers() function inside UsersListViewModel.

    func viewModelCloseList(_ viewModel: UsersListViewModel)
}
